## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| The system answers an unverified hardware or dependency issue | High | Deterministic answer policy blocks experiment/environment categories unless run evidence exists. |
| A code-logic answer is drafted without source evidence | High | Policy requires source evidence before draft eligibility. |
| A simple model bypasses safety through confident proposed text | High | Drafting is downstream of policy; model proposals cannot create drafts directly. |
| Skipped issues are ambiguous to maintainers | Medium | Policy reasons use explicit strings such as `requires_unverified_reproduction` and `missing_source_evidence`. |
</threat_model>
